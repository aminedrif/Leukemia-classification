1. What is Transfer Learning?

Transfer learning reuses a model trained on one task as the starting point for a different task.

Why does it work?
Deep CNNs learn features in layers:

  • Early layers → edges, textures, colors (generic — useful for ANY image task)
  • Middle layers → shapes, patterns, object parts
  • Deep layers → task-specific high-level features

Since early/middle features are universal, we reuse them and only retrain the deep layers for leukemia detection.

Strategy What it does When to use
Feature Extraction Freeze all base layers, train only the new classifier Small dataset, quick baseline
Fine-Tuning Unfreeze some top layers, train with very low learning rate Want higher accuracy

Our approach: Phase 1 (feature extraction) → Phase 2 (fine-tuning).

2. VGG16 Architecture

Paper: "Very Deep Convolutional Networks for Large-Scale Image Recognition" — Simonyan & Zisserman, 2014 (Oxford). Placed 2nd in ILSVRC 2014.

Architecture Diagram

INPUT (224 × 224 × 3) │ ┌─────&boxT;─────┐ │ BLOCK 1 │ Conv2D(64, 3×3) × 2 + MaxPool(2×2) → 112×112×64 &boxcc;────────────&boxb5; │ BLOCK 2 │ Conv2D(128, 3×3) × 2 + MaxPool(2×2) → 56×56×128 &boxcc;────────────&boxb5; │ BLOCK 3 │ Conv2D(256, 3×3) × 3 + MaxPool(2×2) → 28×28×256 &boxcc;────────────&boxb5; │ BLOCK 4 │ Conv2D(512, 3×3) × 3 + MaxPool(2×2) → 14×14×512 &boxcc;────────────&boxb5; │ BLOCK 5 │ Conv2D(512, 3×3) × 3 + MaxPool(2×2) → 7×7×512 ← We fine-tune this └─────&┴;─────┘ │ ┌─────&boxT;──────────────┐ │ OUR CUSTOM HEAD │ ← Replaces original 3 Dense layers │ GlobalAvgPool2D │ 7×7×512 → 512 │ Dense(256, relu) │ │ Dropout(0.5) │ │ Dense(1, sigmoid) │ → Binary output: ALL or HEM └────────────────────┘

Key Design Principles

  • 3×3 filters throughout: Two stacked 3×3 convs = same receptive field as one 5×5, but fewer parameters and more non-linearities
  • Filters double after each pool: 64 → 128 → 256 → 512 (compensates for spatial reduction)
  • ~138 million parameters total (most in the FC layers which we remove)
  • 16 weight layers (13 conv + 3 FC) — hence "VGG16"

Why GlobalAveragePooling2D instead of Flatten?

  • Flatten: 7×7×512 = 25,088 neurons → huge Dense layer → severe overfitting
  • GlobalAveragePooling2D: Averages each feature map → only 512 neurons → much less overfitting

3. Notebook 1: With Augmentation MAIN

File: VGG16-TransferLearning.ipynb

Cell 1 — Imports

import tensorflow as tf
from tensorflow.keras.applications import VGG16
  • TensorFlow/Keras: Our deep learning framework.
  • VGG16: The pre-trained model already trained on ImageNet (1.2M images, 1000 classes).
  • ImageDataGenerator: Loads images and applies real-time augmentation.
  • classification_report, confusion_matrix: Evaluate precision, recall, and errors.

Cell 2 — Dataset Paths

The C-NMC dataset is organized into 3 folds:

  • fold_0 → Training: The model learns from these images.
  • fold_1 → Validation: Used to monitor performance during training. The model does NOT learn from these.
  • fold_2 → Test: Final evaluation ONLY. Never seen during training.

Cell 3 — Data Generators (WITH Augmentation)

What is rescaling? Pixel values go from [0,255] → [0,1]. Neural networks train better with small normalized values.

What is augmentation? Random transformations applied to training images:

Augmentation Value Why it makes sense for cells
rotation_range=20 Rotate ±20° Cells appear at any angle under microscope
width/height_shift=0.1 Shift 10% Cells aren't always centered in the image
horizontal_flip/vertical=True Flip both ways Cells have no inherent orientation
zoom_range=0.1 Zoom ±10% Simulates slightly different magnification levels
Important: We ONLY augment training data. Validation and test must reflect real-world conditions for honest evaluation.

Cell 5 — Build Model (Phase 1)

  • weights='imagenet': Load knowledge about edges, textures, shapes learned from 1.2M images.
  • include_top=False: Remove the original ImageNet classifier.
  • base_model.trainable = False: Freeze ALL 14.7 million parameters.
model = Sequential([
    base_model,                    # VGG16 feature extractor (frozen)
    GlobalAveragePooling2D(),       # 7x7x512 -> 512
    Dense(256, activation='relu'),  # Our hidden layer
    Dropout(0.5),                   # Prevents overfitting
    Dense(1, activation='sigmoid')  # Output probability
])

We only have ~131K trainable parameters out of 14.7M total (< 1%).

Cell 6-8 — Phase 1 Training

  • learning_rate=1e-3 (0.001): Good default for training new layers from scratch.
  • binary_crossentropy: Standard loss function for 2-class classification.

Cell 9-12 — Phase 2 Fine-Tuning

base_model.trainable = True
for layer in base_model.layers[:-4]:
    layer.trainable = False

We unfreeze ONLY Block 5 (the last 4 layers). Block 5 contains high-level features that we want to adapt specifically to leukemia cells.

CRITICAL: We recompile with learning_rate=1e-5. This is 100× smaller! A high LR would destroy the pre-trained features.

Cell 13-16 — Evaluation

Evaluates on fold_2. We generate a Confusion Matrix and Classification Report.

  • Precision: "Of all cells I predicted as ALL, how many actually were ALL?"
  • Recall: "Of all actual ALL cells, how many did I catch?" (Most important in medical AI)

5. What are Callbacks and Why Do We Need Them?

Callbacks are functions that automatically execute at specific points during training. They act like autopilot controls.

1. EarlyStopping

Watches val_loss. If it stops improving for 5 epochs, training STOPS automatically and rewinds to the best epoch.

Why? Prevents overfitting. Without it, the model memorizes training data and gets worse on test data.

2. ReduceLROnPlateau

If val_loss gets stuck for 3 epochs, it halves (0.5x) the learning rate.

Why? Helps the model make smaller, more precise updates to escape plateaus.

3. ModelCheckpoint

Saves the model to disk automatically whenever val_accuracy improves.

Why? If Kaggle crashes or times out, you don't lose your work.

6. Notebook 3: Without Callbacks

File: VGG16-NoCallbacks.ipynb

This notebook removes all three callbacks and trains for a fixed 10 epochs per phase.

Aspect With Callbacks (Notebook 1) Without Callbacks (Notebook 3)
Training duration Stops early when optimal Runs ALL 10 epochs forcefully
Overfitting Prevented High risk after optimal epoch
Learning rate Auto-reduces when stuck Stays fixed
Final model The BEST epoch is saved The LAST epoch is kept (often worse)

7. Suggestions for Better Accuracy

Based on the initial InceptionV3 results (Train 95%, Val 70% → severe overfitting):

  1. Use two-phase training: All our provided VGG16 notebooks do this. Do not train all layers at once initially.
  2. Lower the learning rate: The previous InceptionV3 used 1e-4 for fine-tuning. We use 1e-5.
  3. Freeze more layers: Unfreeze only the last few layers (like Block 5 in VGG16). Unfreezing 111 layers on a small dataset guarantees overfitting.
  4. Use Callbacks: Never train without EarlyStopping.
  5. Try EfficientNet-B3 Next: It offers much better accuracy-to-parameter ratios for medical imaging compared to VGG16 or InceptionV3.
