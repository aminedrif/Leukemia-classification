1. Why InceptionV3?

Paper: "Rethinking the Inception Architecture" — Szegedy et al., 2015 (Google). Successor to GoogLeNet (Inception v1) which won ILSVRC 2014.

The core question InceptionV3 answers: "Why choose one filter size when you can use all of them?"

VGG16 uses only 3×3 filters everywhere. But different features exist at different scales — a small cell nucleus might be best captured by a 1×1 filter, while the overall cell shape needs a 5×5 filter. InceptionV3 processes the input with multiple filter sizes in parallel and concatenates the results.

2. The Inception Module — Multi-Scale Processing

The Core Innovation

Input Feature Map | +----------+----------+----------+----------+ | | | | | Conv 1×1 Conv 1×1 Conv 1×1 MaxPool 3×3 | | | | | Conv 3×3 Conv 5×5 Conv 1×1 | | | | +----------+----------+----------+ | Concatenate (depth) | Output Feature Map
Key insight: Each branch sees the same input but at a different scale. The 1×1 branch captures fine details, the 3×3 captures medium features, and the 5×5 captures broader patterns. The network learns which scale is most useful for each region of the image.

InceptionV3 Improvements over v1

Factorized 5×5Replaced 5×5 with two stacked 3×3 convolutionsSame receptive field, 28% fewer parameters

InceptionV3's advantage for medical imaging: The 299×299 input size means more detail is preserved from the microscopy images. The multi-scale inception modules can simultaneously analyze fine cellular structures (1×1) and overall cell morphology (5×5), which is ideal for distinguishing cancer cells.

4. Notebook 1: With Augmentation MAIN

File: InceptionV3-TransferLearning.ipynb

Key Difference: Input Size 299×299

IMG_SIZE = (299, 299)   # NOT 224×224!
base_model = InceptionV3(weights='imagenet', include_top=False, input_shape=(299, 299, 3))

This is the most important difference. InceptionV3 was designed for 299×299 input. Using 224×224 would lose information in the early layers and hurt accuracy.

Fine-Tuning Strategy: From mixed7 Onward

for i, layer in enumerate(base_model.layers):
    if layer.name == 'mixed7':
        fine_tune_from = i
        break
for layer in base_model.layers[:fine_tune_from]:
    layer.trainable = False

We unfreeze from mixed7 onward — the last 3 inception blocks plus the reduction block. This gives us ~80 trainable layers in the base model. InceptionV3 has many more layers than VGG16, so we can afford to unfreeze more while keeping the early generic features frozen.
