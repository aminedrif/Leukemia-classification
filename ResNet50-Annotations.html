1. Why ResNet50?

Paper: "Deep Residual Learning for Image Recognition" — He et al., 2015 (Microsoft Research). Won 1st place in ILSVRC 2015 with 3.57% top-5 error.

The key problem ResNet solves: the vanishing gradient problem.

Before ResNet, deeper networks (20+ layers) actually performed worse than shallower ones. As the network gets deeper, gradients (the signals used to update weights) become extremely small — they "vanish" before reaching the early layers. This means the early layers stop learning.

ResNet's solution: skip connections (residual connections). Instead of forcing every layer to learn something new, ResNet lets each block learn the difference (residual) between input and output. If a layer has nothing useful to add, it can simply pass the input through unchanged.

2. ResNet50 Architecture — The Skip Connection

The Residual Block (The Core Innovation)

Input (x) | +------ SKIP CONNECTION (identity shortcut) ------+ | | Conv2D 1x1 (reduce channels) | | | BatchNorm + ReLU | | | Conv2D 3x3 (process features) | | | BatchNorm + ReLU | | | Conv2D 1x1 (expand channels) | | | BatchNorm | | | +------------- ADD (x + F(x)) --------------------+ | ReLU | Output = ReLU(x + F(x))
Key insight: The output is ReLU(x + F(x)) where x is the original input (via skip connection) and F(x) is what the conv layers learned. If the conv layers learn nothing useful, F(x) ≈ 0 and the output is simply x — the input passes through unchanged. This is why deeper ResNets never perform worse than shallower ones.

Full Architecture

INPUT (224 × 224 × 3) | Conv2D 7×7, stride 2 → 112×112×64 BatchNorm + ReLU MaxPool 3×3, stride 2 → 56×56×64 | ┌────┴────┐ │ conv2_x │ 3 residual blocks × [64, 64, 256] → 56×56×256 ├─────────┤ │ conv3_x │ 4 residual blocks × [128, 128, 512] → 28×28×512 ├─────────┤ │ conv4_x │ 6 residual blocks × [256, 256, 1024] → 14×14×1024 ├─────────┤ │ conv5_x │ 3 residual blocks × [512, 512, 2048] → 7×7×2048 ← We fine-tune this └────┬────┘ | ┌────┴──────────────────┐ │ OUR CUSTOM HEAD │ │ GlobalAvgPool2D │ 7×7×2048 → 2048 │ BatchNormalization │ Stabilizes training │ Dense(256, relu) │ │ Dropout(0.5) │ │ Dense(1, sigmoid) │ → Binary: ALL or HEM └────────────────────────┘

The Bottleneck Design (1×1 → 3×3 → 1×1)

Each residual block uses a "bottleneck" pattern:

  1. 1×1 Conv (reduce): Shrinks the number of channels (e.g., 256 → 64). This reduces computation for the expensive 3×3 conv.
  2. 3×3 Conv (process): The actual spatial feature extraction happens here, but on fewer channels.
  3. 1×1 Conv (expand): Expands channels back (64 → 256) to match the skip connection.

This "squeeze and expand" trick makes ResNet50 computationally efficient despite being 50 layers deep.

3. VGG16 vs ResNet50 — Key Differences

Feature VGG16 ResNet50
Depth 16 layers 50 layers (3× deeper)
Parameters 138M (14.7M without top) 25.6M (23.5M without top)
Key innovation Uniform 3×3 filters Skip connections (residual)
BatchNormalization ❌ Not used ✅ After every Conv layer
Gradient flow Can vanish in deep nets Always flows via skip connections
Feature vector size 512 (after GAP) 2048 (after GAP) — 4× richer
Fine-tuning target Block 5 (last 4 layers) conv5_x (last ~10 layers)
Expected accuracy ~78-82% ~82-88% (typically higher)
Why ResNet50 should do better: The 2048-dimensional feature vector captures richer cell details than VGG16's 512. Plus, BatchNormalization throughout the network makes training more stable and less sensitive to learning rate choices.

4. Notebook 1: With Augmentation MAIN

File: ResNet50-TransferLearning.ipynb

Cell 1 — Imports

Same as VGG16, but we import ResNet50 instead of VGG16, and we also import BatchNormalization for our custom head.

Cell 2 — Dataset Paths

Same 3-fold split: fold_0 (train), fold_1 (val), fold_2 (test).

Cell 3 — Data Generators (WITH Augmentation)

Identical to VGG16 Notebook 1. Same augmentation parameters (rotation ±20°, flips, zoom ±10%).

Cell 5 — Build Model

base_model = ResNet50(weights='imagenet', include_top=False, input_shape=(224, 224, 3))
base_model.trainable = False

model = Sequential([
    base_model,
    GlobalAveragePooling2D(),   # 7×7×2048 → 2048
    BatchNormalization(),       # NEW vs VGG16!
    Dense(256, activation='relu'),
    Dropout(0.5),
    Dense(1, activation='sigmoid')
])
New: BatchNormalization() — We added this layer between GAP and Dense. It normalizes the 2048 features to have mean=0 and std=1 before feeding them to our Dense layer. This makes training faster and more stable, especially important since ResNet50's feature vectors are 4× larger than VGG16's.

Cell 7 — Unfreezing for Fine-Tuning

fine_tune_at = len(base_model.layers) - 10  # Last ~10 layers
for layer in base_model.layers[:fine_tune_at]:
    layer.trainable = False

We unfreeze the last ~10 layers (the conv5_block3 residual block). This is the last complete residual block in ResNet50. It contains the highest-level features that are the most task-specific.

Cells 8-14 — Same structure as VGG16

Phase 2 training with lr=1e-5, evaluation, confusion matrix, classification report, visual predictions, save model.

8. Suggestions for Better Accuracy

🔴 High Impact — Do These First

# Technique Details Expected Gain
1 Try EfficientNet-B3 Uses compound scaling — simultaneously optimizes depth, width, and resolution. Only 12M params but often outperforms ResNet50. Best accuracy-per-parameter ratio in the literature. +3-8%
2 Label Smoothing Change labels from hard (0 or 1) to soft (0.05 or 0.95). Prevents overconfidence. Code: loss=tf.keras.losses.BinaryCrossentropy(label_smoothing=0.1) +1-3%
3 Class Weighting If ALL and HEM have different counts, the model biases toward the majority class. Fix: calculate weights and pass class_weight={0: w0, 1: w1} to model.fit() +1-4%
4 Cosine Annealing LR Instead of a fixed LR reduced on plateau, smoothly decrease the LR following a cosine curve. Code: tf.keras.optimizers.schedules.CosineDecay(initial_lr, decay_steps) +1-2%

🟡 Medium Impact — Worth Trying

# Technique Details Expected Gain
5 Stronger Augmentation Add brightness_range=[0.8, 1.2], shear_range=0.1, and channel_shift_range=20. Simulates staining variations in microscopy. +1-3%
6 Test-Time Augmentation (TTA) During prediction, create 5-10 augmented copies of each test image, predict all of them, and average the predictions. Free accuracy for no extra training. +1-2%
7 Progressive Fine-Tuning Instead of unfreezing everything at once: unfreeze conv5 → train 5 epochs → unfreeze conv4 → train 5 more. Gradually adapts from top to bottom. +1-3%
8 Mixup / CutMix Advanced augmentation that blends two images together (Mixup) or cuts and pastes patches (CutMix). Very effective for medical imaging. +1-3%
9 Warm-Up Learning Rate Start with a very small LR for the first 2-3 epochs, then gradually increase to the target LR. Prevents early training instability. +0.5-1%

🟢 Advanced — For Maximum Accuracy

# Technique Details Expected Gain
10 Ensemble of Models Train VGG16 + ResNet50 + EfficientNet separately. Average their predictions. Almost always gives the best result. +2-5%
11 K-Fold Cross Validation Instead of a fixed train/val/test split, rotate which fold is used for validation. Train 3 models (one per fold) and average. More reliable accuracy estimate. More robust
12 Attention Mechanisms Add SE (Squeeze-and-Excitation) blocks or CBAM after the base model. These help the network focus on the most important regions of the cell image. +1-3%
13 Stain Normalization Medical images have color variations due to different staining protocols. Normalizing stain colors before training makes the model more robust. +1-4%
14 Mixed Precision Training Use tf.keras.mixed_precision.set_global_policy('mixed_float16'). Trains 2× faster on modern GPUs with almost no accuracy loss. 2× speed
15 Gradient Accumulation Simulate larger batch sizes (64, 128) by accumulating gradients over multiple steps. Larger effective batch sizes can stabilize training. More stable
